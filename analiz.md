# Волны Эллиотта + графические фигуры (объединённая база бота)

Источник EW: [Практическое руководство по волнам Эллиотта на примере BTC/USD](https://tradingsworld.com/base/prakticheskoe-rukovodstvo-po-volnam-elliotta-na-primere-btc-usd/)

Код: `bot/elliott_wave.py`, отрисовка `bot/chart_elliott_draw.py`, интеграция через `wave_structure` → TA → gate → chart.

---

## 1. Восьмиволновой паттерн

- Импульс: волны **1, 3, 5** (по тренду)
- Коррекция внутри импульса: **2, 4**
- После 5 → коррекция **A-B-C** (A/C импульсные против тренда, B — откат)
- Фрактальность: 5-волновка младшего ТФ = одна волна старшего

## 2. Чек-лист правильности (импульс)

1. Волна 2 **не** заходит за основание волны 1
2. Волна 3 — **самая длинная** из 1/3/5 и превышает вершину/дно волны 1
3. Волны 1 и 5 **примерно равны**
4. Волна 4 **не пересекает** территорию волны 1
5. Волна B **не обновляет** основание волны A
6. ABC **не обновляет** основание всей 5-волновки

Дополнительно: чередование резкой/боковой коррекции в волнах 2 и 4.

## 3. Fib-пропорции (ориентиры)

| Волна | Типичный Fib |
|-------|----------------|
| 2 | 0.382–0.618 × 1 |
| 3 | 1.618–2.618 × 1 |
| 4 | 0.382–0.50 × 3 |
| 5 | 0.382–0.618 × 3 |
| A | 0.50–0.618 × 5 |
| B | 0.382–0.50 × A |
| C | часто **1.272 / 1.618 × B** |

## 4. Точки входа (как в боте)

### Консервативный
- После волны 2 (или ABC): вход при **обновлении хая/лоя волны 1**
- После волны 4: вход при **пробое хая/лоя волны 3** → волна 5
- Стоп за основанием волны 1 / за волной 4
- TP ≈ **1:3** к риску

### Агрессивный
- Лимит у **1.272 или 1.618 × волны B** (конец C)
- Стоп **3×ATR**, TP **3×ATR** (R:R ≈ 1:3)
- Подтверждение: пин-бар / поглощение / внутренний бар (в тексте тезиса)

Локация gate: `elliott` (между `fib` и `abc`).

## 5. Графические фигуры (взвешенные факты)

Двойная/тройная вершина·дно, ГиП, флаг/вымпел, треугольники, клинья, прямоугольник,
ложный пробой, 1-2-3, Баскервили, три индейца, ромб, расходящийся треугольник.

Правила из материалов (обязательны для ENTRY по `pattern`):
- **Клин:** обе линии в одну сторону; восх. → чаще SHORT; нисх. → чаще LONG; контекст тренда (прод./разв.)
- **Флаг:** шток + полотнище против тренда; вход после close за линией; стоп за экстремум полотнища; TP ≈ высота штока
- **Вымпел:** шток + сжатие; объём ↓ внутри, ↑ на пробое
- **Тройное дно:** prior downtrend + 3≈равных лоя + пробой сопротивления; объём часто ↓ к 3-му дну; цель = высота; вход breakout или retest

Без confirmed / без объёма на пробое — локация `pattern` ужесточается. Thesis добавляет psychology_note + CVD/liq из factor_lines.

## 7. Pro confluence (максимум по методике)

Код: `bot/setup_confluence.py` → поля `ta.setup_*` / `ta.htf_elliott_*`.

Скоринг 0–100 → грейд **A/B/C/D**:
- HTF Elliott (1h) + LTF EW
- ABCDE / ending diagonal (клин + волна 5)
- графическая фигура
- Fib confluence + SMC
- штраф при конфликте HTF↔LTF

Идеальный вход (`setup_ideal_ready`) → локация gate **`confluence`** (рядом с `elliott`).
Бонус/штраф в `score_trade_setup` через `confluence_boosts_gate`.

Честный лимит: EW субъективен на глаз; в **сигналах** бот торгует только когда
чек-лист + классические Fib-коридоры совпали (см. §3). Цель — отрабатывать
гайд, а не «рисовать красивые цифры».

## 3c. PPT / Prechter — расширения модели (в боте)

Источник-ориентир: презентация Elliott (ppt-online / Prechter): растяжения, усечение, диагонали, типы коррекций.

| Структура | Правило бота | На графике |
|-----------|--------------|------------|
| **Растяжение 1 / 3 / 5** | Самая длинная импульсная ≥ ~1.618× соседней; чаще растяжение **3** | Жёлтая толстая линия + «растяжение N» |
| **Усечение волны 5** | После сильной 3 (≥1.618×1) волна 5 **не** обновляет экстремум 3 — допустимо | Красный ✕ + «усечение 5» |
| **Конечная / начальная диагональ** | Клин + часто перекрытие 4↔1; ending у волны 5, leading у волны 1 | Пунктир границ клина + подпись |
| **ABC зигзаг / плоская** | B≈38–78%A → zigzag; B≥88%A → flat | Бейдж «ABC зигзаг/плоская» |
| **ABCDE треугольник** | Lite в confluence (`is_abcde`) | Метки A–E + тип triangle |

Поля TA: `elliott_extension`, `elliott_truncated`, `elliott_diagonal`, `elliott_corr_type`, `elliott_structure_note`.

## 3d. Полный PPT (https://ppt-online.org/574890) — алгоритмы бота

Код: `bot/elliott_advanced.py` + отрисовка в `chart_elliott_draw.py`.

| Слайд | Алгоритм | На графике / прогноз |
|-------|----------|----------------------|
| Горизонт. △ ABCDE | Схождение A–C / B–D, E внутри | Красные границы + стрелка пробоя по тренду до △ |
| Расходящ. △ | Хаи↑ лои↓; в позиции 4 → стремительная 5 | Мегафон + bias LONG/SHORT |
| W-X-Y / W-X-Y-X-Z | Боковые «двойные/тройные тройки» | Метки WXY(Z) + возобновление тренда |
| Чередование 2/4 | Простая ↔ сложная | Усиливает quality импульса |
| Fib растяжение 3 | w5 ≈ **1.00 × w1** от конца 4 | Горизонт. цель `w5=1.00×w1` |
| Fib растяжение 5 | w5 ≈ **1.618 × (0→3)** от 4 | Цель `w5=1.618×(0→3)` |
| Fib растяжение 1 | 3→5 ≈ **0.618 × w1** | Цель `3→5=0.618×w1` |
| Доля хода | 0→3 = 61.8% всего; 0→4 = 38.2% | Цели `total_0618` / `total_0382` |

## 3e. Два масштаба на графике (глобальный + локальный)

Бот ищет и рисует **оба** слоя, если они есть:
- **G (глобальный):** 1–5 / A–E / ABCDE на всём импульсе (толстая синяя линия)
- **L (локальный):** i–v / a–e внутри последних ~6ч (зелёный пунктир)

Поля: `elliott_global_draw_points`, `elliott_local_draw_points`, `elliott_global_label`, `elliott_local_label`.
