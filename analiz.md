# Волны Эллиотта + графические фигуры (объединённая база бота)

Источник EW: [Практическое руководство по волнам Эллиотта на примере BTC/USD](https://tradingsworld.com/base/prakticheskoe-rukovodstvo-po-volnam-elliotta-na-primere-btc-usd/)

Код: `bot/elliott_wave.py`, отрисовка `bot/chart_elliott_draw.py`, интеграция через `wave_structure` → TA → gate → chart.

---

## 1. Восьмиволновой паттерн

- Импульс: волны **1, 3, 5** (по тренду)
- Коррекция внутри импульса: **2, 4**
- После 5 → коррекция **A-B-C** (A/C импульсные против тренда, B — откат)
- Фрактальность: 5-волновка младшего ТФ = одна волна старшего

## 2. Чек-лист правильности (импульс)

1. Волна 2 **не** заходит за основание волны 1
2. Волна 3 — **самая длинная** из 1/3/5 и превышает вершину/дно волны 1
3. Волны 1 и 5 **примерно равны**
4. Волна 4 **не пересекает** территорию волны 1
5. Волна B **не обновляет** основание волны A
6. ABC **не обновляет** основание всей 5-волновки

Дополнительно: чередование резкой/боковой коррекции в волнах 2 и 4.

## 3. Fib-пропорции (ориентиры)

| Волна | Типичный Fib |
|-------|----------------|
| 2 | 0.382–0.618 × 1 |
| 3 | 1.618–2.618 × 1 |
| 4 | 0.382–0.50 × 3 |
| 5 | 0.382–0.618 × 3 |
| A | 0.50–0.618 × 5 |
| B | 0.382–0.50 × A |
| C | часто **1.272 / 1.618 × B** |

## 4. Точки входа (как в боте)

### Консервативный
- После волны 2 (или ABC): вход при **обновлении хая/лоя волны 1**
- После волны 4: вход при **пробое хая/лоя волны 3** → волна 5
- Стоп за основанием волны 1 / за волной 4
- TP ≈ **1:3** к риску

### Агрессивный
- Лимит у **1.272 или 1.618 × волны B** (конец C)
- Стоп **3×ATR**, TP **3×ATR** (R:R ≈ 1:3)
- Подтверждение: пин-бар / поглощение / внутренний бар (в тексте тезиса)

Локация gate: `elliott` (между `fib` и `abc`).

## 5. Графические фигуры (уже в боте, ~12 семейств)

Двойная вершина/дно, ГиП / перевёрнутая, флаг/вымпел, треугольники (симм./возр./нисх.), клинья, прямоугольник, ложный пробой, 1-2-3, Баскервили, три индейца, ромб, расходящийся треугольник.

**Объединение:** паттерн + EW + Fib/SMC усиливают друг друга. ENTRY только при локации (`fib` / `elliott` / `abc` / `pattern` / `retest` / `sr`) и score ≥ порога.

## 6. Отрисовка на графике

- Точки и линии **0-1-2-3-4-5** (синий, LTF)
- **A-B-C** / **A-B-C-D-E** (оранжевый)
- **HTF** волны — пунктир (перенос с 1h на индекс LTF)
- Прогнозный путь Pro: entry → path → tp (+ inv пунктир)
- Пунктир зоны EW-входа + стоп
- Fib 0.382/0.5/0.618 и фигуры — как раньше

## 7. Pro confluence (максимум по методике)

Код: `bot/setup_confluence.py` → поля `ta.setup_*` / `ta.htf_elliott_*`.

Скоринг 0–100 → грейд **A/B/C/D**:
- HTF Elliott (1h) + LTF EW
- ABCDE / ending diagonal (клин + волна 5)
- графическая фигура
- Fib confluence + SMC
- штраф при конфликте HTF↔LTF

Идеальный вход (`setup_ideal_ready`) → локация gate **`confluence`** (рядом с `elliott`).
Бонус/штраф в `score_trade_setup` через `confluence_boosts_gate`.

Честный лимит: EW субъективен; бот ловит **класс** сетапа (HTF+фигура+зона), не ручную разметку 1:1.

Иерархия на графике (с v48+):
1. Вердикт → только путь этой стороны (WAIT = без tp-стрелок)
2. Fib на доминирующей ноге (дамп > микро-отскок)
3. Микро EW в боковике не рисуем
4. Боковик/post-crash без пробоя → WAIT ≤6/10
