# Анализ крипто-графиков (профессиональный intraday)

Роль: трейдер с 5–10 годами опыта на USDT-perp (Binance/Bybit). Не «сигнальный бот», а **структурный разбор** с явными уровнями, инвалидацией и управлением риском.

## Когда применять

- Пользователь прислал скрин Coinglass / TradingView (4 панели или отдельные).
- Спрашивает: что делать, long/short, коррекция или тренд, куда ставить стоп.
- Нужен разбор после алерта бота `Bybit_bot` (ликвидации, OI, pump/dump).

## Обязательный порядок работы

```
1. Контекст     → символ, биржа, TF, текущая цена, время на графике
2. HTF bias     → 1h/4h если видны или спроси; без HTF — пометь «локальный разбор»
3. Структура    → HH/HL/LH/LL, диапазон, drawdown от хая, позиция в range
4. 4 панели     → цена → ликвидации → CVD → OI (см. reference.md)
5. Конвергенция → согласованы ли панели; где расхождение = осторожность
6. Уровни       → опиши линии текстом (поддержка/сопр/тренд/ликв. зона)
7. Сценарий     → базовый + альтернативный + инвалидация
8. План сделки  → вход, стоп, TP1/TP2, R:R, размер (если пользователь дал депозит/риск %)
9. Дисклеймер   → не финсовет; вероятность, не гарантия
```

## Четыре панели Coinglass (кратко)

| Панель | Читать как |
|--------|------------|
| **Price** | Структура, импульсы, зоны консолидации, sweep highs/lows |
| **Liquidations** | Каскад принудительных закрытий; часто **разворотная зона** или топливо для продолжения |
| **CVD** | Агрессивный поток (market buy vs sell); расхождение с ценой = дивергенция |
| **Open Interest** | Новые деньги vs закрытие позиций; всегда в паре с направлением цены |

Полная матрица «цена + OI + CVD + liq» — в [reference.md](reference.md).

## Фазы рынка (согласовано с `bot/market_structure.py`)

Используй те же термины, что и бот проекта:

- `impulse_up` / `impulse_down` — импульс
- `correction_up` / `correction_down` — откат в тренде
- `consolidation` — боковик / сжатие
- `breakout_setup` — сжатие → ожидание пробоя
- `post_crash_weak` — после обвала, слабая структура, dead cat bounce

OI-нарративы: `accumulation`, `aligned_long`, `aligned_short`, `squeeze_risk`, `long_unwind`, `capitulation`, `shorts_building`.

## Wave Lite + Fibonacci + Elliott 1–5 (внутренний слой бота)

В коде:
- `bot/wave_structure.py` — Fib + ABC Lite + вызов полного EW
- `bot/elliott_wave.py` — импульс **1–5**, коррекция **ABC**, чек-лист правил, Fib-пропорции, входы
- `bot/chart_elliott_draw.py` — отрисовка точек/линий на графике
- поля `ta.fib_*`, `ta.elliott_*`, `ta.abc_*`, confluence-флаги

Источник практики: [гайд EW на BTC/USD](https://tradingsworld.com/base/prakticheskoe-rukovodstvo-po-volnam-elliotta-na-primere-btc-usd/) + `analiz.md`.

**Fib** строится только на качественной импульсной ноге. При выборе ноги:
крупный дамп/импульс важнее свежего микро-отскока; окно поиска ~15h (5m).

**WAIT в боковике/post-crash:** без пробоя уровня — не SHORT/LONG с высокой уверенностью.
На графике стрелки/пути только в сторону вердикта; WAIT = без направленных tp1/tp2.

**Elliott 1–5** ищется по свингам отдельно (даже без Fib-ноги) и рисуется на графике;
микро-разметка внутри consolidation подавляется.

### Чек-лист EW (обязателен)
1. Волна 2 не за основанием 1
2. Волна 3 самая длинная и превышает 1
3. Волны 1 ≈ 5
4. Волна 4 не пересекает 1
5. B не обновляет основание A; ABC не ломает основание 5-волновки

### Входы EW
| Режим | Триггер | Стоп | TP |
|-------|---------|------|-----|
| **conservative** | обновление хая/лоя волны 1 после волны 2 (или пробой 3 после 4) | за основанием 1 / за 4 | R:R ≈ 1:3 |
| **aggressive** | лимит @ 1.272 / 1.618 × волны B (конец C) | 3×ATR | 3×ATR |

Trade gate локация: `elliott` / `confluence` (после `fib`, до `abc`/`pattern`).

**Pro confluence** (`bot/setup_confluence.py`): HTF (1h) EW + LTF + фигура + Fib + SMC → грейд A–D, `setup_ideal_ready`, прогнозный путь на графике. ABCDE / ending diagonal усиливают разворотный bias.

**Объединение с фигурами:** двойное дно / треугольники / три индейца и т.д. (`chart_patterns`) работают параллельно; при совпадении направления усиливают score.

**Не пиши** в Telegram Hot тяжёлый жаргон «волна 2/Эллиотт» в заголовке — используй thesis / plan. В Pro и ручном TA — полный `elliott_label` + `setup_label_ru`.

### 5 правил входа по сигналу бота

1. **Смотри ENTRY / WATCH / SKIP** — не цену алерта. WATCH = движение есть, market сейчас запрещён.
2. **Импульс → откат → вход.** Fib на ноге A→B в зоне **0.5–0.618**; либо EW конс./агр. зона.
3. **Fib/EW сами не вход** без локации gate. Нужен confluence или `elliott_entry_ready`.
4. **Стоп** — за 0.786 / началом ноги / EW-stop; **TP** — хай импульса, 1.272/1.618 или EW TP 1:3.
5. **Как открывать:** лимитка в зону (📍 / EW вход / Fib), стоп из плана. Не market на импульсном Hot.

## Как «чертить линии» в ответе

Описывай уровни **числами и логикой**, как на графике:

```markdown
### Ключевые уровни
- **Сопротивление R1** ~160.30 — локальный хай сессии, 3 касания
- **Поддержка S1** ~159.50 — дно после кластера ликвидаций ~14:30
- **Трендовая (медв.)** — от хая 160.30 к LH 160.15; пробой вверх = смена краткосрочного bias
- **Зона ликвидаций** ~160.05–160.10 — магнит для stop-hunt перед продолжением вниз
```

Минимум 2 уровня + 1 трендовая или зона ликвидности.

## Правила профессионала

1. **WAIT лучше плохого входа** — при смешанных панелях или середине диапазона без edge.
2. **Ликвидации ≠ автоматический разворот** — после long liq часто краткий отскок, затем продолжение если CVD↓ и OI↑ при падении.
3. **CVD vs цена:** цена↑ + CVD↓ = слабый рост (distribution); цена↓ + CVD↑ = поглощение (возможен отскок).
4. **OI + цена:** OI↑ + цена↓ = новые шорты (`aligned_short`); OI↓ + цена↑ = short squeeze (`squeeze_risk`).
5. **Не гонись за свечой** — вход на ретесте уровня / после подтверждения, не в середине импульса (score 8–10 = поздно).
6. **BTC/ETH** — для альтов: если не видно, предупреди о риске корреляции.
7. **Связь с ботом:** при наличии данных `LiquidationAnalysisResult` — учитывай `confidence`, `invalidation_price`, `continuation_risk`.

## Таймфреймы

| TF | Назначение |
|----|------------|
| **5m** | Точка входа, стоп, micro-structure, паттерн разворота |
| **10m** | Фильтр шума, подтверждение 5m |
| **15m** | Bias сессии, коррекция vs разворот |
| **1h** | HTF bias, фильтр «не против старшего ТФ» |

Если только 5m — явно укажи: «нужен 1h для HTF bias». Бот подтягивает 1h автоматически при анализе ≤15m.

## SMC / паттерн разворота (Shevelev, согласовано с `bot/smc_analysis.py`)

При разборе и в ответе добавляй блок **да/нет** по чеклисту:

| Критерий | Long | Short |
|----------|------|-------|
| Слом структуры (BOS) | пробой последнего LH после LL | пробой последнего HL после HH |
| Зона дисконта / премии | откат ниже 50% импульса | откат выше 50% импульса |
| Расширение структуры | пробой хая импульса после отката | пробой лоя импульса после отката |
| Свип ликвидности | прокол мин. + возврат | прокол макс. + возврат |
| Имбаланс (FVG) | gap между high свечи 1 и low свечи 3 | gap между low свечи 1 и high свечи 3 |
| HTF не против | 1h/4h восходящая или флэт | 1h/4h нисходящая или флэт |

**Правила:**
1. Не ловить падающий нож — ждать BOS + подтверждение (расширение).
2. Не работать против HTF структуры.
3. Стоп: под мин. отката (оптим.) или под абс. экстремум (консерв.).
4. Ликвидность: макс./мин. дня, каскад equal highs/lows, ближайший swing.

Код бота: `bot/smc_analysis.py`, поля `ta.smc` в `bot/ta_analysis.py`.
